# AGENTS.md

## Cursor Cloud specific instructions

This is a Next.js 16 personal portfolio site using **Bun** as the package manager and runtime.

### Key commands

| Task | Command |
|------|---------|
| Install deps | `bun install` |
| Dev server | `bun dev` (port 3000) |
| Lint | `bun run lint` |
| Build | `bun run build` |
| Fetch data | `bun run fetch-data` |

### Data files

Two JSON data files (`src/data/tweets.json` and `src/data/github-stars.json`) are gitignored and generated by `bun run fetch-data`. Before running `bun dev` for the first time, run `bun run fetch-data` to create them. Without API tokens, tweets will be empty and GitHub stars will use the public API (rate-limited but functional).

### External services (all optional)

All external services degrade gracefully when unconfigured:
- **Upstash Redis** (`UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`): traffic tracking; returns zeros when absent.
- **Twitter API** (`TWITTER_API_BEARER_TOKEN`): tweet carousel; hidden when absent.
- **GitHub API** (`GITHUB_TOKEN`): star counts; uses public API without token.
- **Vercel Analytics**: no-op outside Vercel.

### Gotchas

- The project uses `bun.lock` (not `package-lock.json` / `yarn.lock` / `pnpm-lock.yaml`). Always use `bun install`, not npm/yarn/pnpm.
- Next.js 16 with Turbopack is used for both dev and build. No special flags needed.
- There are no automated tests in this repository; lint (`bun run lint`) is the primary code quality check.
